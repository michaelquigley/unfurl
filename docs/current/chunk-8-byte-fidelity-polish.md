# Chunk 8 — Byte-Fidelity Edge Cases and Polish

Implemented final byte-fidelity behavior:

- `internal/tokenize.Document` now records detected line-ending discipline from the first non-empty line.
- Reflowed paragraph output uses the document-level LF/CRLF discipline instead of each source segment's original ending.
- Tokenization records whether the input ended with a newline; reflow preserves absence of a trailing newline when the final paragraph segment had none.
- UTF-8 BOM strip-and-reattach behavior is covered for plain paragraphs, YAML front matter, and ATX headings.
- CRLF paragraph and hard-break output is covered.
- Mixed line endings in a reflowed paragraph now follow the first non-empty line's detected discipline.
- Unclosed fenced code blocks are covered and pass through to EOF.
- The property corpus includes generated byte-fidelity fixtures for BOM, CRLF, and no-trailing-newline cases.
- README and AGENTS.md were updated from implementation-in-progress wording to current behavior.

Verification for the completed implementation is `make test`, which runs unit tests, CLI tests, property/idempotence tests, and `go vet ./...`.
